    25тЖТ    25тЖТ    25тЖТ

   429тЖТ   429тЖТ   429тЖТ## 5. Major Problems Faced & Resolutions
   430тЖТ   430тЖТ   430тЖТ
   431тЖТ   431тЖТ   431тЖТ### 5.1 CRITICAL: Infinite Recursion in use-api-data.ts
   432тЖТ   432тЖТ   432тЖТ
   433тЖТ   433тЖТ   433тЖТ**Problem:** `const fetch = useCallback(async () => { const res = await fetch(url) })` тАФ the inner `fetch(url)` called itself recursively instead of the global `fetch`. This broke ALL API data fetching in GMR tab silently (no error thrown, just infinite recursion that eventually timed out).
   434тЖТ   434тЖТ   434тЖТ
   435тЖТ   435тЖТ   435тЖТ**Resolution:** Renamed to `fetchData` and used `globalThis.fetch` for the actual network call. Also replaced all `any` types with `Record<string, unknown>`.
   436тЖТ   436тЖТ   436тЖТ
   437тЖТ   437тЖТ   437тЖТ**Impact:** GMR model data was completely broken until this fix. This was a silent failure тАФ no error boundary caught it.
   438тЖТ   438тЖТ   438тЖТ
   439тЖТ   439тЖТ   439тЖТ### 5.2 CRITICAL: Stale Closure in LiveActivityFeed
   440тЖТ   440тЖТ   440тЖТ
   441тЖТ   441тЖТ   441тЖТ**Problem:** `useState(0)` tick counter was included in `useEffect` dependency array, causing the interval to be destroyed and recreated every 3 seconds (on every tick). This caused memory leaks and visual flickering.
   442тЖТ   442тЖТ   442тЖТ
   443тЖТ   443тЖТ   443тЖТ**Resolution:** Replaced with `useRef(0)` tickRef, removed `tick` from dependency array.
   444тЖТ   444тЖТ   444тЖТ
   445тЖТ   445тЖТ   445тЖТ### 5.3 CRITICAL: Interval Leaks in StressLab Dialogs
   446тЖТ   446тЖТ   446тЖТ
   447тЖТ   447тЖТ   447тЖТ**Problem:** `RunTestDialog` and `BatchRunDialog` created `setInterval` timers for progress simulation but never cleaned them up on component unmount. If the user closed the dialog mid-test, the interval kept running forever.
   448тЖТ   448тЖТ   448тЖТ
   449тЖТ   449тЖТ   449тЖТ**Resolution:** Added `useRef<ReturnType<typeof setInterval>>()` for interval refs, cleanup on unmount via useEffect return, clear via ref instead of captured closure.
   450тЖТ   450тЖТ   450тЖТ
   451тЖТ   451тЖТ   451тЖТ### 5.4 HIGH: SVG Gradient ID Collision in Charts
   452тЖТ   452тЖТ   452тЖТ
   453тЖТ   453тЖТ   453тЖТ**Problem:** Multiple `MiniAreaChart` instances with `dataKey="value"` used the same gradient ID `grad-value`. When multiple charts rendered on the same page, they shared gradients, causing color bleeding тАФ one chart's gradient would override another's.
   454тЖТ   454тЖТ   454тЖТ
   455тЖТ   455тЖТ   455тЖТ**Resolution:** Added `useId()` from React to generate unique IDs: `grad-${uid}-${dataKey}`. Applied to both `MiniAreaChart` and `NexusStackedAreaChart`.
   456тЖТ   456тЖТ   456тЖТ
   457тЖТ   457тЖТ   457тЖТ### 5.5 HIGH: Hydration Mismatch in Clock Components
   458тЖТ   458тЖТ   458тЖТ
   459тЖТ   459тЖТ   459тЖТ**Problem:** Server renders `--:--:--` but client renders the actual time on first paint, causing React hydration mismatch warnings and potential UI flickering.
   460тЖТ   460тЖТ   460тЖТ
   461тЖТ   461тЖТ   461тЖТ**Resolution:** Created `useMounted` hook using `useSyncExternalStore` (later simplified to `useState('--:--:--')` + `useEffect` pattern) so both server and client render the same placeholder initially.
   462тЖТ   462тЖТ   462тЖТ
   463тЖТ   463тЖТ   463тЖТ### 5.6 HIGH: AI Assistant Double-Message Bug
   464тЖТ   464тЖТ   464тЖТ
   465тЖТ   465тЖТ   465тЖТ**Problem:** `addChatMessage` was called before the API call, so when `useNexusStore.getState().chatMessages` was read for the API body, it already included the user message. The API then also appended the user message, causing duplication.
   466тЖТ   466тЖТ   466тЖТ
   467тЖТ   467тЖТ   467тЖТ**Resolution:** Capture `currentMessages` from store BEFORE adding user message, then use `currentMessages + user message` in API call body.
   468тЖТ   468тЖТ   468тЖТ
   469тЖТ   469тЖТ   469тЖТ### 5.7 HIGH: Dynamic Tailwind Class Bugs
   470тЖТ   470тЖТ   470тЖТ
   471тЖТ   471тЖТ   471тЖТ**Problem:** Using template literals like `bg-${t.color}-600/15` for Tailwind classes тАФ Tailwind's JIT compiler can't detect dynamic classes, so they were purged in production builds, resulting in unstyled elements.
   472тЖТ   472тЖТ   472тЖТ
   473тЖТ   473тЖТ   473тЖТ**Resolution:** Replaced ALL dynamic class strings with explicit conditional classes using ternary operators. Found in both `vault-tab.tsx` and `governor-tab.tsx` (and later `rate-limit-tab.tsx`).
