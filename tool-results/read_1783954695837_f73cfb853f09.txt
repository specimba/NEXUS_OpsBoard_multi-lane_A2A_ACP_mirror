    24‚Üí## 1. Executive Summary
    25‚Üí
    26‚ÜíNEXUS OS Command Center is an 8-pillar AI governance dashboard built with Next.js 16 + Turbopack on the frontend and a Python governance backend (`nexus_os/`) with 67 modules and 586+ pytest tests. The dashboard features real-time WebSocket updates, AI-powered assistant chat, transparent model routing (AI Provider Bridge), and interactive trust governance controls.
    27‚Üí
    28‚Üí**Key Metrics:**
    29‚Üí- **Frontend:** 107 TypeScript files, 16,814 lines across 26 custom components
    30‚Üí- **Backend (Python):** 67 modules across 9 packages, 586 tests (2 collection errors)
    31‚Üí- **API Routes:** 19 endpoints (10 GET + 7 POST + 2 PUT)
    32‚Üí- **Database:** SQLite via Prisma ORM with 12 models
    33‚Üí- **Real-time:** WebSocket mini-service on port 3003 (Socket.io)
    34‚Üí- **Lint Status:** 0 errors, 0 warnings
    35‚Üí- **Design:** 8 interconnected tab modules, dark/light theme, responsive mobile

   429‚Üí## 5. Major Problems Faced & Resolutions
   430‚Üí
   431‚Üí### 5.1 CRITICAL: Infinite Recursion in use-api-data.ts
   432‚Üí
   433‚Üí**Problem:** `const fetch = useCallback(async () => { const res = await fetch(url) })` ‚Äî the inner `fetch(url)` called itself recursively instead of the global `fetch`. This broke ALL API data fetching in GMR tab silently (no error thrown, just infinite recursion that eventually timed out).
   434‚Üí
   435‚Üí**Resolution:** Renamed to `fetchData` and used `globalThis.fetch` for the actual network call. Also replaced all `any` types with `Record<string, unknown>`.
   436‚Üí
   437‚Üí**Impact:** GMR model data was completely broken until this fix. This was a silent failure ‚Äî no error boundary caught it.
   438‚Üí
   439‚Üí### 5.2 CRITICAL: Stale Closure in LiveActivityFeed
   440‚Üí
   441‚Üí**Problem:** `useState(0)` tick counter was included in `useEffect` dependency array, causing the interval to be destroyed and recreated every 3 seconds (on every tick). This caused memory leaks and visual flickering.
   442‚Üí
   443‚Üí**Resolution:** Replaced with `useRef(0)` tickRef, removed `tick` from dependency array.
   444‚Üí
   445‚Üí### 5.3 CRITICAL: Interval Leaks in StressLab Dialogs
   446‚Üí
   447‚Üí**Problem:** `RunTestDialog` and `BatchRunDialog` created `setInterval` timers for progress simulation but never cleaned them up on component unmount. If the user closed the dialog mid-test, the interval kept running forever.
   448‚Üí
   449‚Üí**Resolution:** Added `useRef<ReturnType<typeof setInterval>>()` for interval refs, cleanup on unmount via useEffect return, clear via ref instead of captured closure.
   450‚Üí
   451‚Üí### 5.4 HIGH: SVG Gradient ID Collision in Charts
   452‚Üí
   453‚Üí**Problem:** Multiple `MiniAreaChart` instances with `dataKey="value"` used the same gradient ID `grad-value`. When multiple charts rendered on the same page, they shared gradients, causing color bleeding ‚Äî one chart's gradient would override another's.
   454‚Üí
   455‚Üí**Resolution:** Added `useId()` from React to generate unique IDs: `grad-${uid}-${dataKey}`. Applied to both `MiniAreaChart` and `NexusStackedAreaChart`.
   456‚Üí
   457‚Üí### 5.5 HIGH: Hydration Mismatch in Clock Components
   458‚Üí
   459‚Üí**Problem:** Server renders `--:--:--` but client renders the actual time on first paint, causing React hydration mismatch warnings and potential UI flickering.
   460‚Üí
   461‚Üí**Resolution:** Created `useMounted` hook using `useSyncExternalStore` (later simplified to `useState('--:--:--')` + `useEffect` pattern) so both server and client render the same placeholder initially.
   462‚Üí
   463‚Üí### 5.6 HIGH: AI Assistant Double-Message Bug
   464‚Üí
   465‚Üí**Problem:** `addChatMessage` was called before the API call, so when `useNexusStore.getState().chatMessages` was read for the API body, it already included the user message. The API then also appended the user message, causing duplication.
   466‚Üí
   467‚Üí**Resolution:** Capture `currentMessages` from store BEFORE adding user message, then use `currentMessages + user message` in API call body.
   468‚Üí
   469‚Üí### 5.7 HIGH: Dynamic Tailwind Class Bugs
   470‚Üí
   471‚Üí**Problem:** Using template literals like `bg-${t.color}-600/15` for Tailwind classes ‚Äî Tailwind's JIT compiler can't detect dynamic classes, so they were purged in production builds, resulting in unstyled elements.
   472‚Üí
   473‚Üí**Resolution:** Replaced ALL dynamic class strings with explicit conditional classes using ternary operators. Found in both `vault-tab.tsx` and `governor-tab.tsx` (and later `rate-limit-tab.tsx`).
   474‚Üí
   475‚Üí### 5.8 MEDIUM: Tooltip Import Collisions
   476‚Üí
   477‚Üí**Problem:** Both `recharts` and `@/components/ui/tooltip` export a `Tooltip` component. When both were imported in the same file, the second import silently overrode the first, breaking either chart tooltips or UI tooltips.
   478‚Üí
   479‚Üí**Resolution:** Consistently alias recharts Tooltip to `RechartsTooltip` (or `RechartsTooltipComponent`) across all affected files (tokens-tab.tsx, stresslab-tab.tsx, gmr-tab.tsx, governor-tab.tsx, vault-tab.tsx).
   480‚Üí
   481‚Üí### 5.9 MEDIUM: PieChart Naming Collision
   482‚Üí
   483‚Üí**Problem:** Both `lucide-react` and `recharts` export `PieChart`. In `tokens-tab.tsx`, the lucide icon import silently overrode the recharts component.
   484‚Üí
   485‚Üí**Resolution:** Renamed lucide import to `PieChartIcon` (or `PieChartLucide` in vault-tab.tsx).
   486‚Üí
   487‚Üí### 5.10 MEDIUM: Light Mode Text Contrast
   488‚Üí
   489‚Üí**Problem:** All colored text used `-400` Tailwind shades (designed for dark backgrounds). In light mode, these were nearly invisible ‚Äî emerald-400 on white, red-400 on white, etc.
   490‚Üí
   491‚Üí**Resolution:** Bulk-replaced 200+ instances across 18 files: `text-{color}-400` ‚Üí `text-{color}-600 dark:text-{color}-400`. Added 12 light-theme CSS overrides for tables, glows, badges, scrollbars.
   492‚Üí
   493‚Üí### 5.11 OPERATIONAL: Dev Server Process Persistence
   494‚Üí
   495‚Üí**Problem:** Background processes die when the Bash tool shell exits. The dev server needs to be manually restarted between operations.
   496‚Üí
   497‚Üí**Resolution:** Partial ‚Äî used `bun run dev` with `tee dev.log` for monitoring. Cron job handles auto-restart during review cycles. Still a known issue for manual operations.
