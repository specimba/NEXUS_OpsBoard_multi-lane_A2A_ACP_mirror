   212→   212→D. MCP INTEGRATION ARCHITECTURE
   213→   213→================================
   214→   214→
   215→   215→The MCP (Model Context Protocol) access layer is defined across three files:
   216→   216→
   217→   217→1. bridge/mcpaauth.py — RBAC + API key authentication
   218→   218→   - 4 roles: ADMIN / OPERATOR / READONLY / GUEST
   219→   219→   - 6 AuthResults: OK / INVALID_KEY / EXPIRED / RATE_LIMITED / FORBIDDEN / NOT_FOUND
   220→   220→   - APIKey dataclass: key_id, key_value, role, agent_id, created_at, expires_at, rate_limit (declared but UNUSED), request_count
   221→   221→   - Permission map (method-based): tasks/submit->tasks:write, tasks/status->tasks:read, vault/write->vault:write, vault/read->vault:read, a2a/agent-card->a2a:read
   222→   222→   - Role permissions (with wildcard support): ADMIN={*}, OPERATOR={tasks:*, vault:*, a2a:*}, READONLY={tasks:read, vault:read, status:*}, GUEST={status:read}
   223→   223→   - 3 default keys hardcoded: admin-001, operator-001, readonly-001
   224→   224→   - Singleton via get_mcp_auth() with double-checked locking (threading.Lock)
   225→   225→   - GAP: rate_limit field is declared but authenticate() never enforces it
   226→   226→   - GAP: mcpaauth.py is NOT imported by bridge/server.py — two parallel auth systems exist
   227→   227→
   228→   228→2. bridge/server.py — JSON-RPC 2.0 governance server
   229→   229→   - BridgeServer wraps FastAPI via create_app()
   230→   230→   - 5 methods: tasks/submit, tasks/status, vault/read, vault/write, a2a/agent-card (A2A v1.1)
   231→   231→   - Endpoints: POST /tasks/submit, POST /tasks/status, POST /vault/read, POST /vault/write, POST / (JSON-RPC router), GET /health
   232→   232→   - Auth: HMAC-SHA256 signature via X-Nexus-Signature header (uses SecretStore, NOT MCPAuth API keys)
   233→   233→   - Authz: KAIJU 4-variable (scope x clearance, impact x clearance, intent x action) via NexusGovernor.check_access()
   234→   234→   - Token guard: pre-check 1000-token budget -> 429 if exceeded; tracks input/output tokens via TokenGuard; non-blocking on failure
   235→   235→   - JSON-RPC errors: -32700 ParseError, -32600 Method Not Allowed, -32601 Unknown Method, -32602 Invalid Params, -32603 Internal Error; HeldError -> HTTP 202 with hold_ticket
   236→   236→   - Response envelope: {jsonrpc: "2.0", result, trace_id, x-nexus-input-tokens, x-nexus-output-tokens}
   237→   237→
   238→   238→3. bridge/sdk.py — NexusClient client SDK
   239→   239→   - HMAC-SHA256 request signing: signature = HMAC-SHA256(secret, "{secret}:{trace_id}:{payload}")
   240→   240→   - X-Nexus-* headers: Agent-ID, Project-ID, Trace-ID, Signature, Lineage-ID (optional)
   241→   241→   - RetryPolicy: max_retries=3, base_delay=1.0s, max_delay=30s, backoff_factor=2.0, retryable_codes=[429, 500, 502, 503, 504]
   242→   242→   - CircuitBreaker: states CLOSED/OPEN/HALF_OPEN, failure_threshold=5, recovery_timeout=60s, thread-safe via threading.Lock
   243→   243→   - Public API: submit_task, query_status, read_memory, write_memory
   244→   244→   - Uses urllib (not aiohttp) — sync-only by default; async mentioned as optional in docstring
   245→   245→
   246→   246→HOW AN EXTERNAL MCP SYSTEM WOULD CONNECT:
   247→   247→1. Acquire an agent_id + secret from SecretStore (env var NEXUS_SECRET_{AGENT_ID} or secrets.json file or master-key derivation)
   248→   248→2. Register an API key with MCPAuth.register_key() — currently optional since server.py doesn't enforce it
   249→   249→3. POST to http://bridge-host:7352/ with body {"method": "tasks/submit", "description": "...", "context": {...}, "kaiju": {scope, intent, impact, clearance}}
   250→   250→4. Headers required: X-Nexus-Agent-ID, X-Nexus-Project-ID, X-Nexus-Trace-ID, X-Nexus-Signature, Content-Type: application/json
   251→   251→5. Response: JSON-RPC 2.0 envelope with task_id + status + token_usage
   252→   252→
   253→   253→A2A v1.1: query via POST / with method="a2a/agent-card" returns AgentCard with capabilities, trust_band, negotiation_policies, status.    299→   299→   - PRIOR_SUCCESS=10, PRIOR_FAILURE=2 prevent small-sample overfitting

   306→   306→   - Critical complexity -> prefer local models if available above threshold, else quality-first
   307→   307→   - Default -> highest score
   308→   308→   - Fallback: if no candidates above threshold, use highest-score model with "below threshold" reason

   310→   310→9. DAG-BASED TASK ROUTING (EngineRouter):

   315→   315→   - Self-loop detection: parent_task_id == child_task_id rejected
   316→   316→   - Status states: PENDING, IN_PROGRESS, COMPLETED, FAILED, BLOCKED, CANCELLED
