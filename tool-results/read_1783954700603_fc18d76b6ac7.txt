    24→    24→## 1. Executive Summary
    25→    25→
    26→    26→NEXUS OS Command Center is an 8-pillar AI governance dashboard built with Next.js 16 + Turbopack on the frontend and a Python governance backend (`nexus_os/`) with 67 modules and 586+ pytest tests. The dashboard features real-time WebSocket updates, AI-powered assistant chat, transparent model routing (AI Provider Bridge), and interactive trust governance controls.
    27→    27→
    28→    28→**Key Metrics:**
    29→    29→- **Frontend:** 107 TypeScript files, 16,814 lines across 26 custom components
    30→    30→- **Backend (Python):** 67 modules across 9 packages, 586 tests (2 collection errors)
    31→    31→- **API Routes:** 19 endpoints (10 GET + 7 POST + 2 PUT)
    32→    32→- **Database:** SQLite via Prisma ORM with 12 models
    33→    33→- **Real-time:** WebSocket mini-service on port 3003 (Socket.io)
    34→    34→- **Lint Status:** 0 errors, 0 warnings
    35→    35→- **Design:** 8 interconnected tab modules, dark/light theme, responsive mobile

   429→   429→## 5. Major Problems Faced & Resolutions
   430→   430→
   431→   431→### 5.1 CRITICAL: Infinite Recursion in use-api-data.ts
   432→   432→
   433→   433→**Problem:** `const fetch = useCallback(async () => { const res = await fetch(url) })` — the inner `fetch(url)` called itself recursively instead of the global `fetch`. This broke ALL API data fetching in GMR tab silently (no error thrown, just infinite recursion that eventually timed out).
   434→   434→
   435→   435→**Resolution:** Renamed to `fetchData` and used `globalThis.fetch` for the actual network call. Also replaced all `any` types with `Record<string, unknown>`.
   436→   436→
   437→   437→**Impact:** GMR model data was completely broken until this fix. This was a silent failure — no error boundary caught it.
   438→   438→
   439→   439→### 5.2 CRITICAL: Stale Closure in LiveActivityFeed
   440→   440→
   441→   441→**Problem:** `useState(0)` tick counter was included in `useEffect` dependency array, causing the interval to be destroyed and recreated every 3 seconds (on every tick). This caused memory leaks and visual flickering.
   442→   442→
   443→   443→**Resolution:** Replaced with `useRef(0)` tickRef, removed `tick` from dependency array.
   444→   444→
   445→   445→### 5.3 CRITICAL: Interval Leaks in StressLab Dialogs
   446→   446→
   447→   447→**Problem:** `RunTestDialog` and `BatchRunDialog` created `setInterval` timers for progress simulation but never cleaned them up on component unmount. If the user closed the dialog mid-test, the interval kept running forever.
   448→   448→
   449→   449→**Resolution:** Added `useRef<ReturnType<typeof setInterval>>()` for interval refs, cleanup on unmount via useEffect return, clear via ref instead of captured closure.
   450→   450→
   451→   451→### 5.4 HIGH: SVG Gradient ID Collision in Charts
   452→   452→
   453→   453→**Problem:** Multiple `MiniAreaChart` instances with `dataKey="value"` used the same gradient ID `grad-value`. When multiple charts rendered on the same page, they shared gradients, causing color bleeding — one chart's gradient would override another's.
   454→   454→
   455→   455→**Resolution:** Added `useId()` from React to generate unique IDs: `grad-${uid}-${dataKey}`. Applied to both `MiniAreaChart` and `NexusStackedAreaChart`.
   456→   456→
   457→   457→### 5.5 HIGH: Hydration Mismatch in Clock Components
   458→   458→
   459→   459→**Problem:** Server renders `--:--:--` but client renders the actual time on first paint, causing React hydration mismatch warnings and potential UI flickering.
   460→   460→
   461→   461→**Resolution:** Created `useMounted` hook using `useSyncExternalStore` (later simplified to `useState('--:--:--')` + `useEffect` pattern) so both server and client render the same placeholder initially.
   462→   462→
   463→   463→### 5.6 HIGH: AI Assistant Double-Message Bug
   464→   464→
   465→   465→**Problem:** `addChatMessage` was called before the API call, so when `useNexusStore.getState().chatMessages` was read for the API body, it already included the user message. The API then also appended the user message, causing duplication.
   466→   466→
   467→   467→**Resolution:** Capture `currentMessages` from store BEFORE adding user message, then use `currentMessages + user message` in API call body.
   468→   468→
   469→   469→### 5.7 HIGH: Dynamic Tailwind Class Bugs
   470→   470→
   471→   471→**Problem:** Using template literals like `bg-${t.color}-600/15` for Tailwind classes — Tailwind's JIT compiler can't detect dynamic classes, so they were purged in production builds, resulting in unstyled elements.
   472→   472→
   473→   473→**Resolution:** Replaced ALL dynamic class strings with explicit conditional classes using ternary operators. Found in both `vault-tab.tsx` and `governor-tab.tsx` (and later `rate-limit-tab.tsx`).
   474→   474→
   475→   475→### 5.8 MEDIUM: Tooltip Import Collisions
   476→   476→
   477→   477→**Problem:** Both `recharts` and `@/components/ui/tooltip` export a `Tooltip` component. When both were imported in the same file, the second import silently overrode the first, breaking either chart tooltips or UI tooltips.
   478→   478→
   479→   479→**Resolution:** Consistently alias recharts Tooltip to `RechartsTooltip` (or `RechartsTooltipComponent`) across all affected files (tokens-tab.tsx, stresslab-tab.tsx, gmr-tab.tsx, governor-tab.tsx, vault-tab.tsx).
   480→   480→
   481→   481→### 5.9 MEDIUM: PieChart Naming Collision
   482→   482→
   483→   483→**Problem:** Both `lucide-react` and `recharts` export `PieChart`. In `tokens-tab.tsx`, the lucide icon import silently overrode the recharts component.
   484→   484→
   485→   485→**Resolution:** Renamed lucide import to `PieChartIcon` (or `PieChartLucide` in vault-tab.tsx).
   486→   486→
   487→   487→### 5.10 MEDIUM: Light Mode Text Contrast
   488→   488→
   489→   489→**Problem:** All colored text used `-400` Tailwind shades (designed for dark backgrounds). In light mode, these were nearly invisible — emerald-400 on white, red-400 on white, etc.
   490→   490→
   491→   491→**Resolution:** Bulk-replaced 200+ instances across 18 files: `text-{color}-400` → `text-{color}-600 dark:text-{color}-400`. Added 12 light-theme CSS overrides for tables, glows, badges, scrollbars.
   492→   492→
   493→   493→### 5.11 OPERATIONAL: Dev Server Process Persistence
   494→   494→
   495→   495→**Problem:** Background processes die when the Bash tool shell exits. The dev server needs to be manually restarted between operations.
   496→   496→
   497→   497→**Resolution:** Partial — used `bun run dev` with `tee dev.log` for monitoring. Cron job handles auto-restart during review cycles. Still a known issue for manual operations.
