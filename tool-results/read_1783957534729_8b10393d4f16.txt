   113→   212→D. MCP INTEGRATION ARCHITECTURE
   114→   213→================================
   115→   214→
   116→   215→The MCP (Model Context Protocol) access layer is defined across three files:
   117→   216→
   118→   217→1. bridge/mcpaauth.py — RBAC + API key authentication
   119→   218→   - 4 roles: ADMIN / OPERATOR / READONLY / GUEST
   120→   219→   - 6 AuthResults: OK / INVALID_KEY / EXPIRED / RATE_LIMITED / FORBIDDEN / NOT_FOUND
   121→   220→   - APIKey dataclass: key_id, key_value, role, agent_id, created_at, expires_at, rate_limit (declared but UNUSED), request_count
   122→   221→   - Permission map (method-based): tasks/submit->tasks:write, tasks/status->tasks:read, vault/write->vault:write, vault/read->vault:read, a2a/agent-card->a2a:read
   123→   222→   - Role permissions (with wildcard support): ADMIN={*}, OPERATOR={tasks:*, vault:*, a2a:*}, READONLY={tasks:read, vault:read, status:*}, GUEST={status:read}
   124→   223→   - 3 default keys hardcoded: admin-001, operator-001, readonly-001
   125→   224→   - Singleton via get_mcp_auth() with double-checked locking (threading.Lock)
   126→   225→   - GAP: rate_limit field is declared but authenticate() never enforces it
   127→   226→   - GAP: mcpaauth.py is NOT imported by bridge/server.py — two parallel auth systems exist
   128→   227→
   129→   228→2. bridge/server.py — JSON-RPC 2.0 governance server
   130→   229→   - BridgeServer wraps FastAPI via create_app()
   131→   230→   - 5 methods: tasks/submit, tasks/status, vault/read, vault/write, a2a/agent-card (A2A v1.1)
   132→   231→   - Endpoints: POST /tasks/submit, POST /tasks/status, POST /vault/read, POST /vault/write, POST / (JSON-RPC router), GET /health
   133→   232→   - Auth: HMAC-SHA256 signature via X-Nexus-Signature header (uses SecretStore, NOT MCPAuth API keys)
   134→   233→   - Authz: KAIJU 4-variable (scope x clearance, impact x clearance, intent x action) via NexusGovernor.check_access()
   135→   234→   - Token guard: pre-check 1000-token budget -> 429 if exceeded; tracks input/output tokens via TokenGuard; non-blocking on failure
   136→   235→   - JSON-RPC errors: -32700 ParseError, -32600 Method Not Allowed, -32601 Unknown Method, -32602 Invalid Params, -32603 Internal Error; HeldError -> HTTP 202 with hold_ticket
   137→   236→   - Response envelope: {jsonrpc: "2.0", result, trace_id, x-nexus-input-tokens, x-nexus-output-tokens}
   138→   237→
   139→   238→3. bridge/sdk.py — NexusClient client SDK
   140→   239→   - HMAC-SHA256 request signing: signature = HMAC-SHA256(secret, "{secret}:{trace_id}:{payload}")
   141→   240→   - X-Nexus-* headers: Agent-ID, Project-ID, Trace-ID, Signature, Lineage-ID (optional)
   142→   241→   - RetryPolicy: max_retries=3, base_delay=1.0s, max_delay=30s, backoff_factor=2.0, retryable_codes=[429, 500, 502, 503, 504]
   143→   242→   - CircuitBreaker: states CLOSED/OPEN/HALF_OPEN, failure_threshold=5, recovery_timeout=60s, thread-safe via threading.Lock
   144→   243→   - Public API: submit_task, query_status, read_memory, write_memory
   145→   244→   - Uses urllib (not aiohttp) — sync-only by default; async mentioned as optional in docstring
   146→   245→
   147→   246→HOW AN EXTERNAL MCP SYSTEM WOULD CONNECT:
   148→   247→1. Acquire an agent_id + secret from SecretStore (env var NEXUS_SECRET_{AGENT_ID} or secrets.json file or master-key derivation)
   149→   248→2. Register an API key with MCPAuth.register_key() — currently optional since server.py doesn't enforce it
   150→   249→3. POST to http://bridge-host:7352/ with body {"method": "tasks/submit", "description": "...", "context": {...}, "kaiju": {scope, intent, impact, clearance}}
   151→   250→4. Headers required: X-Nexus-Agent-ID, X-Nexus-Project-ID, X-Nexus-Trace-ID, X-Nexus-Signature, Content-Type: application/json
   152→   251→5. Response: JSON-RPC 2.0 envelope with task_id + status + token_usage
   153→   252→
   154→   253→A2A v1.1: query via POST / with method="a2a/agent-card" returns AgentCard with capabilities, trust_band, negotiation_policies, status.
