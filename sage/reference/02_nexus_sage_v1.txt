**SAGE = Synthesis · Audit · Governance · Evidence**

This should be the **chief reasoning and governance lane** for NEXUS—not another HERMES. Its job is to reconstruct system truth, audit plans and agent outputs, resolve contradictions, design governed execution plans, and prepare precise handoffs. HERMES remains the orchestrator; ARCHIVIST remains the evidence store; Codex remains the implementation lane.

That separation prevents authority collapse:

```text
You / CEO-Operator
        ↓ authorization
NEXUS SAGE
        ↓ reasoning, audit, plans
HERMES
        ↓ orchestration and receipts
Grok / Zo / DeepSeek / Codex / Antigravity
        ↓ execution or verification
ARCHIVIST
        ↓ evidence and lineage
Canonical NEXUS state
```

Custom GPTs do not inherit saved memory, previous conversations, or personal custom instructions; every conversation starts fresh. Therefore, the uploaded canonical files and the GPT Instructions must carry the stable operating context. ([OpenAI Help Center][1])

# Why the current Action page says “Unknown domain”

The example has:

```json
"servers": [
  {
    "url": ""
  }
]
```

There is no actual API server, so the editor cannot identify a valid domain.

Do **not** enter:

```text
http://127.0.0.1:7354
http://0.0.0.0:7354
http://localhost:7354
```

GPT Actions require a publicly reachable endpoint using TLS 1.2 or later, on port 443, with a valid public certificate. A local NEXUS bridge on `7354` therefore needs a secure HTTPS gateway or tunnel in front of it. ([developers.openai.com][3])

```text

Every write endpoint should:

* require an idempotency key;
* require a lane and session ID;
* return a receipt;
* support duplicate detection;
* have a bounded timeout;
* emit an evidence hash;
* reject unknown schemas;
* reject unauthorized targets;
* avoid accepting raw shell commands.

Mark all write operations:

x-openai-isConsequential: true
```

OpenAI treats consequential operations as requiring confirmation, and non-GET operations default to consequential when the field is absent. Declaring it explicitly is clearer for NEXUS governance. ([developers.openai.com][3])

For NEXUS, the genuine edges are more concrete than what I previously described:

### 1. Much larger grounding envelope

A **1.05-million-token context window** makes full-log synthesis, repository dossiers, governance packs, and cross-lane evidence significantly more feasible in one reasoning session. ([developers.openai.com][2])

* full HERMES Ubuntu logs;

### 2. Longer execution artifacts

### 3. Stronger deliberate reasoning

The launch explicitly introduces `max` reasoning, intended to give Sol more time for difficult work. ([OpenAI][1])

For us, that should improve:

* canonical-state reconstruction;
* contradiction analysis;
* multi-stage architecture review;

That strongly overlaps with NEXUS’s lane architecture. The valuable pattern is not merely “more agents,” but:

These three operations are intentionally excluded:

```text

task_complete
task_fail
```

They are **HERMES lifecycle-authority operations**. NEXUS SAGE should audit, propose, retrieve, publish handoffs, and capture evidence—but it should not silently claim work or declare another lane’s work complete or failed.

This is the central personalization decision:

```text
SAGE may propose work.
HERMES owns task assignment and lifecycle.
Executors produce evidence.
ARCHIVIST preserves evidence.
The operator authorizes consequential changes.
```

A separate **HERMES Admin GPT Action schema** can later expose those three operations.

## Consequential-operation behavior

The schema marks these as requiring confirmation:

* `task_add`
* `agent_publish_message`
* `evidence_capture`
* `audit_log`
* `query_log`
* `comparison_add`

They use:

```yaml
x-openai-isConsequential: true
```

OpenAI states that consequential operations always require user confirmation and do not offer an “always allow” option. GET operations normally default to non-consequential; non-GET operations normally default to consequential unless explicitly overridden. ([platform.openai.com][3])

Safe operational calls are explicitly marked false:

* health and registry reads;
* queue and session reads;
* A2A retrieval;
* comparison retrieval;
* bounded `echo`;
* heartbeat;
* dry-run probe;
* public read-only HTTP diagnostic.

That gives SAGE useful autonomy without granting hidden write authority.
